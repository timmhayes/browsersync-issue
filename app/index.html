Browsersync iframe Issue

Issue

When hosted by Browsersync, <a> tags with a <target> attribute linking to an <iframe> element now open in new tab. They should open in the linked <iframe> instead.
Ex: <a href="page1.html" target="main">load page 1</a>

Expected behavior

The links below should force page 1 and 2 to be shown within the iframe content. Run the content locally to verify.

Observed behavior in Browsersync

Links pop open in new tabs. This was not true of previous Browsersync versions.
